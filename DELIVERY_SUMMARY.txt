================================================================================
  🔌 BACKEND REQUIREMENTS
================================================================================

Two API endpoints need to be implemented:

1. GET /api/lectures/{lectureId} (ENHANCED)
   - Return slides array with alternatives
   - Each slide needs:
     * content (text)
     * selectedFlowchart (object with id, url, label)
     * selectedImage (object with id, url, label)
     * flowcharts array (alternatives)
     * images array (alternatives)

2. POST /api/lectures/{lectureId}/selections (NEW)
   - Accept selections object mapping slide indices to selected IDs
   - Save to database
   - Return updated lecture object

See CODE_EXAMPLES.md for exact request/response formats.
